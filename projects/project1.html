AI Document Analyzer

Full-stack semantic search and retrieval-augmented generation pipeline for PDFs. Designed to let users upload research papers and ask natural language questions, receiving accurate answers with citations.

Problem

Traditional keyword search often fails when phrasing differs or context spans multiple sections. The goal was to build an AI-powered analyzer that enables natural queries over large academic PDFs.

Approach

  • Parsing & Chunking: Used PyMuPDF to split PDFs into ~500-token chunks with metadata (doc_id, page_num).
  • Embeddings: Generated sentence embeddings with Hugging Face all-MiniLM-L6-v2.
  • Vector Search: Stored embeddings in FAISS, with metadata persisted in SQLite.
  • Reranking: Applied a cross-encoder (ms-marco-MiniLM-L-6-v2) to improve precision.
  • RAG: Passed top chunks to an LLM (OpenAI/HF) for grounded answers with citations.
  • Stack: FastAPI backend, React + Tailwind frontend.

Results

  • ~92% top-3 retrieval accuracy across 100+ academic PDFs.
  • Average query latency < 1.2s (ANN + reranking).
  • Tested by ~20 peers; strong feedback on clarity of citations.

Reflection

This project strengthened my skills in vector search systems, RAG pipelines, and full-stack AI integration. It also highlighted the importance of safeguards to reduce hallucinations and improve reliability.
